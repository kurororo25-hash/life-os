# データ構造

localStorage に保存するデータの構造をまとめています。

---

## 共通ルール

- すべての項目は `id`（一意な文字列）と `createdAt`（ISO 8601形式）を持つ
- 更新したものは `updatedAt` が追加される
- `Storage.add()` を使うと `id` と `createdAt` が自動付与される

---

## タイムボクシング（`life_timebox`）

```json
{
  "id": "abc123",
  "date": "2026-07-12",
  "title": "資料作成",
  "startTime": "09:00",
  "endTime": "09:30",
  "done": false,
  "gcalEventId": "Googleカレンダーに同期済みの場合のイベントID",
  "createdAt": "...",
  "updatedAt": "..."
}
```

| フィールド | 型 | 値 |
|-----------|----|----|
| date | string | `YYYY-MM-DD`（その予定がある日） |
| startTime / endTime | string | `HH:MM`（24時間表記）。所要時間は常に30分固定（`endTime` = `startTime` + 30分） |
| done | boolean | 完了なら `true` |
| gcalEventId | string | Googleカレンダー同期済みの場合のみ存在 |

タイムライン表示範囲は朝2:00〜夜21:00（`timebox.html` 内の `BASE_HOUR` / `END_HOUR` 定数）。

## リマインダー（`life_reminders`）

```json
{
  "id": "abc123",
  "title": "薬を飲む",
  "date": "2026-07-01",
  "priority": "high",
  "repeat": "daily",
  "done": false,
  "notes": "朝食後",
  "createdAt": "2026-06-29T10:00:00.000Z",
  "updatedAt": "2026-06-29T12:00:00.000Z"
}
```

| フィールド | 型 | 値 |
|-----------|----|----|
| priority | string | `"high"` / `"medium"` / `"low"` |
| repeat | string | `"none"` / `"daily"` / `"weekly"` / `"monthly"` |
| done | boolean | 完了なら `true` |
