# データ構造

localStorage に保存するデータの構造をまとめています。

---

## 共通ルール

- すべての項目は `id`（一意な文字列）と `createdAt`（ISO 8601形式）を持つ
- 更新したものは `updatedAt` が追加される
- `Storage.add()` を使うと `id` と `createdAt` が自動付与される

---

## リマインダー（`life_reminders`）

```json
{
  "id": "abc123",
  "title": "薬を飲む",
  "date": "2026-07-01",
  "priority": "high",
  "repeat": "daily",
  "done": false,
  "notes": "朝食後",
  "createdAt": "2026-06-29T10:00:00.000Z",
  "updatedAt": "2026-06-29T12:00:00.000Z"
}
```

| フィールド | 型 | 値 |
|-----------|----|----|
| priority | string | `"high"` / `"medium"` / `"low"` |
| repeat | string | `"none"` / `"daily"` / `"weekly"` / `"monthly"` |
| done | boolean | 完了なら `true` |

---

## 在庫（`life_inventory`）

```json
{
  "id": "abc123",
  "name": "シャンプー",
  "category": "日用品",
  "unit": "本",
  "currentCount": 1,
  "minCount": 2,
  "location": "洗面台下",
  "notes": "",
  "createdAt": "..."
}
```

`currentCount <= minCount` のとき在庫少アラートが出る。

---

## 買い物リスト（`life_shopping`）

```json
{
  "id": "abc123",
  "name": "シャンプー",
  "quantity": "1本",
  "category": "日用品",
  "source": "inventory",
  "inventoryId": "在庫のID",
  "bought": false,
  "notes": "",
  "createdAt": "..."
}
```

| フィールド | 値 |
|-----------|----|
| source | `"manual"` (手動追加) / `"inventory"` (在庫から自動) |
| inventoryId | source が inventory のとき在庫の id が入る |

## 車内荷物（`life_cargo`）

```json
{
  "id": "abc123",
  "name": "A4クリアファイル",
  "category": "書類",
  "qty": 1,
  "location": "助手席足元",
  "always": "yes",
  "notes": "",
  "createdAt": "..."
}
```

| always | 意味 |
|--------|------|
| `"yes"` | 常時積んでいる |
| `"no"` | 必要時のみ |

## 日記（`life_diary`）

```json
{
  "id": "abc123",
  "date": "2026-06-29",
  "mood": "good",
  "content": "今日は現場がスムーズに終わった。",
  "createdAt": "..."
}
```

| mood | 表示 |
|------|------|
| `"great"` | 😄 最高 |
| `"good"` | 🙂 よかった |
| `"neutral"` | 😐 ふつう |
| `"bad"` | 😔 つらい |
